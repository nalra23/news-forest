# 2. IA Prompt

```prompt
You are a senior system architect and UX-focused product designer collaborating with a CTO, PM, and UX designer.

Your task is to generate a comprehensive Information Architecture (IA) document.

The product is a gamified news platform called "News Forest", targeting the Korean market.

All output must be written in Korean using Markdown.

---

<prd-summary>
News Forest is a web-based gamified news platform designed for users who do not typically consume news.

Core concept:
- Users read news → earn points → grow a tree → build a forest

Key features:
- Article reading validation (scroll + dwell time)
- Point system and growth stages
- AI-based personalized news recommendation
- Anonymous user system (device-based ID)
- Social interaction (watering other users' trees)

Target:
- Gen Z / Millennials
- Users who avoid news consumption

Constraints:
- Web-based MVP
- Anonymous usage
- External news API integration (e.g., Segye Ilbo)
</prd-summary>

<navigation-type>
Mobile:
- Topbar + Bottom navigation

Desktop:
- Topbar + optional Sidebar (no bottom navigation)
</navigation-type>

<auth-type>
Anonymous (device-based ID)
</auth-type>

<responsive-priority>
Mobile-first
</responsive-priority>

<seo-priority>
- High: News article pages (for traffic acquisition)
- Medium: Landing / onboarding pages
- Low/Blocked: Personal forest pages (noindex)
</seo-priority>

---

## Important Instructions

1. If structure is ambiguous:
   - Make reasonable assumptions and mark them as "Note:"

2. Optimize for:
   - Low friction onboarding (no login)
   - Strong engagement loop (reading → reward → growth → social)

3. Anonymous ID policy:
   - Do NOT expose raw device IDs in URLs
   - Use abstracted or hashed identifiers if needed
   - Clearly distinguish between public vs private pages

4. SEO considerations:
   - Clearly separate indexable vs non-indexable pages
   - Reflect this in URL structure

5. Keep architecture MVP-friendly (avoid over-engineering)

---

## IA must include:

- Site Map  

- User Flow  
  (first visit → onboarding → engagement → retention loop)

- Navigation Structure  
  (mobile vs desktop differences must be explicit)

- Page Hierarchy  
  (core vs secondary pages clearly separated)

- Content Organization  

- Interaction Patterns  

- URL Structure  
  (include SEO + anonymous ID policy)

- Component Hierarchy  

---

## Required Core Pages (must be included)

- Home (tree + news feed)
- Article detail (reading experience)
- Tree growth / visualization view
- Social forest map (explore other users)
- Point / activity dashboard
- Onboarding / tutorial

---

## UI State Guidelines

For key pages, define:
- Loading state
- Empty state
- Error state

---

## Accessibility (Optional but recommended)

- Consider Korean Web Accessibility Guidelines (KWCAG)
- Ensure readable contrast and simple interaction flows

---

## Formatting Rules

- Language: Korean
- Markdown format
- Use tables and structured diagrams where appropriate
- Clearly distinguish mobile vs desktop behavior
- Include internal linking strategy
```
