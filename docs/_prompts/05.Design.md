# 5. Design Guide Prompt

```prompt
You are a senior UI/UX designer and frontend design system architect collaborating with a PM, CTO, and frontend engineer.

Your task is to generate a comprehensive Design Guide for "News Forest", a gamified news platform targeting the Korean market.

All output must be written in Korean using Markdown.

---

<product-context>
News Forest is a web-based gamified news platform for users who typically avoid reading news.

Core concept:
- Users read news
- Earn points
- Grow a tree
- Build a forest
- Receive personalized news recommendations
- Interact with other users through watering

The design should make news reading feel light, rewarding, calm, and visually satisfying.
</product-context>

<design-preferences>
Style: modern, friendly, calm, nature-inspired
Color Scheme: soft green / neutral / warm accent

Primary Color: #2F855A
Secondary Color: #A7F3D0
Accent Color: #FBBF24
Base Background: #F8FAF7
Text Color: #1F2937
</design-preferences>

<mood-keywords>
calm, playful, trustworthy, warm, nature, growth, habit-forming, light
</mood-keywords>

<typography>
Font family:
- Primary UI: Pretendard
- Article reading: Pretendard or Noto Serif KR (for long-form readability)
- Numeric display: use tabular-nums for stable point/count animations

Typography roles:
- Heading
- Body
- Caption
- Article reading text
</typography>

<theme-support>
- MVP: Light mode only (defined above)
- Future (Phase 2): Dark mode support
- Dark mode should be optimized for long-form reading (reduced eye strain)
- Use Tailwind `dark:` variant for implementation
</theme-support>

<target-users>
- News-avoiding users
- Gen Z / Millennials
- Users who prefer visual and gamified experiences
</target-users>

<platform>
Web MVP
Responsive priority: Mobile-first

Mobile:
- Topbar + Bottom navigation

Desktop:
- Topbar + expanded layout (no bottom navigation)
</platform>

<core-pages>
- Landing / onboarding page
- Home page with tree and news feed
- Article detail page
- Tree growth visualization page
- Social forest map page
- Point / activity dashboard
- Recommendation feed
</core-pages>

---

## Important Instructions

1. Design should reduce the psychological burden of reading news.
2. Prioritize clarity, readability, and emotional reward.
3. Gamification must feel subtle and trustworthy, not childish.
4. Avoid cluttered UI and excessive animation.
5. Clearly distinguish mobile and desktop layouts.
6. Include accessibility considerations based on WCAG 2.2.
7. If design assumptions are uncertain, mark them as "Note:".

---

## Gamification Design Constraints (Very Important)

To maintain a calm and trustworthy experience:

Avoid:
- Cartoon-style mascots or childish illustrations
- Excessive reward animations (e.g., confetti explosions)
- Highly saturated primary colors used as large backgrounds
- Comic-style fonts or playful exaggerated UI

Prefer:
- Minimal and clean illustration style
- Subtle micro-interactions instead of large animations
- Nature-inspired visual metaphors (growth, leaves, forest)
- Calm and balanced color usage (accent used sparingly)

Reference tone:
- Forest app
- Headspace (subtle delight approach)
- Notion (calm UI baseline)

---

## UI State Tone Guidelines

Ensure all system states match the brand mood:

- Empty state:
  - Gentle, encouraging tone
  - Nature metaphor (e.g., "아직 씨앗이 심어지지 않았어요")

- Loading state:
  - Use skeleton UI
  - Optional subtle plant-themed animation

- Error state:
  - Avoid aggressive red alerts
  - Use warm tone and clear recovery action
  - Example tone: "잠시 문제가 있었어요. 다시 시도해볼까요?"

- Keyboard navigation support
- Readable font size for long-form articles
- Reduced motion option
- Semantic HTML structure
```
