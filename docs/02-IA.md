## 2. URL Structure (SEO + 익명 ID 정책)

### 2.1 URL 명명 규칙

| 패턴 | 사용 케이스 | 예시 |
|------|------------|------|
| 케밥 케이스 | 모든 path 세그먼트 | `/articles/it/ai-trend-2026` |
| Slug | Article path 마지막 세그먼트 | `ai-trend-2026` |
| Public-safe ID | 익명 사용자 노출 식별자 (UUID v4) | `/forest/u/3f8c9d10-...` |

### 2.2 SEO 정책

| 페이지 그룹 | SEO 우선순위 | meta robots | 사유 |
|-------------|-------------|-------------|------|
| `/articles/:category/:slug` | **HIGH** | `index, follow` | 트래픽 유입의 핵심 — News Forest의 SEO 무기 |
| `/articles`, `/articles/:category` | Medium | `index, follow` | 카테고리 hub 페이지 |
| `/` (Landing) | Medium | `index, follow` | 브랜드 검색 + 신규 유입 |
| `/onboarding/*` | Low | `noindex, nofollow` | 실험적 UX, 색인 가치 낮음 |
| `/home`, `/dashboard`, `/forest/me`, `/settings` | None | **`noindex, nofollow`** | 개인화 페이지, 익명 ID 사용 |
| `/forest/u/:publicId` | None | **`noindex, nofollow`** | 익명 사용자 보호 |

### 2.3 익명 ID 노출 금지 정책 (CLAUDE.md §9 준수)

| 식별자 | 저장 위치 | URL 노출 |
|--------|----------|----------|
| Raw device fingerprint | ❌ 어디에도 저장 X | ❌ |
| SHA-256 해시 anonymous ID (내부) | DB 내부 PK | ❌ 절대 노출 X |
| Public-safe UUID v4 | DB의 별도 컬럼 | ✅ `/forest/u/:publicId` 에서만 |

> **Note:** Public-safe UUID는 가입 시 한 번 생성 후 변경되지 않음. 노출되더라도 raw device로 역추적 불가능.

### 2.4 canonical URL

- Article의 정식 URL: `/articles/:category/:slug`
- 카테고리가 다른 경우 canonical은 primary category로 통일 (`<link rel="canonical">`)

## 3. User Flow

### 3.1 First Visit → Onboarding (Cold Entry)

```mermaid
flowchart LR
    A[첫 방문<br>/ 또는 /articles/...] --> B{Anonymous ID 존재?}
    B -- 없음 --> C[Device fingerprint 생성<br>SHA-256 해시 저장]
    C --> D[/onboarding/welcome]
    D --> E[/onboarding/concept]
    E --> F[/onboarding/categories<br>관심 카테고리 ≥1 선택]
    F --> G[/home<br>첫 추천 피드 노출]
    B -- 있음 --> G
```

### 3.2 Engagement Loop (핵심 Reading Cycle)

```mermaid
flowchart LR
    H[/home] --> I[추천 피드에서<br>article card 클릭]
    I --> J[/articles/:category/:slug]
    J --> K{Scroll ≥ 90%<br>AND Dwell ≥ 30s?}
    K -- Yes --> L[+10 포인트<br>잎사귀 모션 1회]
    L --> M{Tree 단계 변화?}
    M -- Yes --> N[차분한 트랜지션<br>Seed→Sprout 등]
    M -- No --> O[다음 추천 카드 표시]
    N --> O
    O --> H
    K -- No --> P[조건 미충족<br>포인트 미지급]
    P --> H
```

### 3.3 Retention Loop (재방문)

```mermaid
flowchart LR
    Q[7일 후 재방문] --> R[Anonymous ID 복구<br>localStorage + 쿠키]
    R --> S[/home]
    S --> T{Streak 정보 표시}
    T --> U[Engagement Loop 재진입]
```

### 3.4 Social Loop (Watering)

```
1. /forest/explore 진입 → 다른 사용자 나무 카드 N개 노출 (랜덤)
2. 카드 탭 → /forest/u/:publicId 이동
3. "물 주기" 버튼 → 24h 중복 검증 → +2P (행위자) + 시각 피드백 (대상)
4. 다시 explore로 돌아가거나 home으로 이동
```

|------|------|

| **Body** | 페이지 콘텐츠 (Bottom nav 높이만큼 padding-bottom) |
| **Bottom nav (fixed)** | Home (🌱) / Forest (🌳) / Dashboard (📊) — 3개 탭 |

```
┌──────────────────────┐
│ Topbar  [logo] [12P] │  ← sticky
├──────────────────────┤
│                      │
│   Page Content       │
│                      │
├──────────────────────┤
│ [Home][Forest][Dash] │  ← fixed bottom
```

| **Main** | 페이지 콘텐츠 (max-width 1200px) |

## 7. Interaction Patterns

### 7.1 Article Reading Progress

| 트리거 | 동작 | 시각 피드백 |
|--------|------|------------|
| 페이지 진입 | Visibility timer 시작 | — |
| Scroll 진행 | 진행률 라인 차오름 | 상단 2px 라인 |
| Tab blur | Timer 일시정지 | 진행률 라인 흐려짐 |
| Tab focus | Timer 재시작 | 라인 복원 |
| 90% + 30s 충족 | `complete` 이벤트 | 잎사귀 1회 모션 (350ms) + 카운트업 |

### 7.2 Point Earning Feedback

- 포인트 카운트업: `tabular-nums`, 350ms ease-out
- Toast: "기사 한 편 읽었어요. +10P" (3s 자동 dismiss)
- 폭죽·confetti 등 과한 연출 **금지** (CLAUDE.md §6.4 / 05-DesignGuide)

### 7.3 Tree Growth Transition

| from → to | 트랜지션 |
|-----------|---------|
| Seed → Sprout | 350ms fade + scale 1.05 → 1.0 |
| Sprout → Tree | 동일 |
| Tree → Forest | 약간 더 길게 (500ms) — 마지막 단계 강조 |

### 7.4 Watering Interaction

```
Tap "물주기" 버튼
  → 24h 중복 체크 (server)
  → 가능하면 물방울 모션 1회 (300ms)
  → +2P 카운트업 (행위자만)
  → 대상자 측은 다음 방문 시 "익명의 누군가가 물을 줬어요" 알림
```

### 7.5 Recommendation Refresh

| 플랫폼 | 트리거 |
|--------|--------|
| Mobile | Pull-to-refresh |
| Desktop | "새로고침" 버튼 클릭 |

### 7.6 Tab/Concurrent Behavior

- 동일 article을 여러 탭에서 열면 **첫 탭만 timer 활성**
- BroadcastChannel API로 탭 간 lock 동기화 (anti-abuse)

## 8. UI State Guidelines (페이지별 3-state)

### 8.1 Home (`/home`)

| State | 화면 | 메시지 톤 |
|-------|------|----------|
| Loading | Article card skeleton 5개 + Tree widget skeleton | — |
| Empty (첫 방문, 추천 미생성) | "첫 기사를 골라볼까요?" + 카테고리 6개 노출 | 격려 |
| Error | "잠시 문제가 있었어요. 다시 시도해볼까요?" + Retry 버튼 | 따뜻함 |

### 8.2 Article Detail (`/articles/:category/:slug`)

| State | 화면 | 메시지 톤 |
|-------|------|----------|
| Loading | 본문 skeleton (3 paragraph) | — |
| Error | "기사를 불러오지 못했어요" + 카테고리 인덱스로 이동 버튼 | 따뜻함 |
| 완독 실패 (이탈) | 별도 화면 없음. 다음 방문 시 "이어서 읽기" 카드 노출 | — |

### 8.3 Forest (`/forest/me`)

| State | 화면 | 메시지 톤 |
|-------|------|----------|
| Loading | Tree skeleton (회색 원) | — |
| Empty (아직 Seed) | "아직 씨앗이 심어지지 않았어요. 첫 기사를 읽어볼까요?" | 격려 + 자연 메타포 |
| Error | "당신의 숲을 불러오지 못했어요" | 따뜻함 |

### 8.4 Dashboard (`/dashboard`)

| State | 화면 |
|-------|------|
| Loading | 통계 카드 skeleton 4개 |
| Empty | "활동 기록이 곧 쌓일 거예요" |
| Error | "통계를 불러오지 못했어요" |

## 10. Internal Linking Strategy

| From | To | 링크 위치 | 목적 |
|------|----|----|------|
| `/articles/:category/:slug` | `/articles/:category` | 본문 상단 breadcrumb | 카테고리 탐색 강화 |
| `/articles/:category/:slug` | 다른 article | 본문 하단 RelatedArticles | 체류 증가 |
| `/home` (Article card) | `/articles/:category/:slug` | 카드 클릭 | 핵심 진입 |
| `/home` (Tree widget) | `/forest/me` | 위젯 클릭 | 시각적 보상 강화 |
| `/forest/explore` (User card) | `/forest/u/:publicId` | 카드 클릭 | 사회적 인터랙션 |
| `/forest/u/:publicId` (Watering) | `/forest/explore` | Watering 후 자동 복귀 | Loop 유도 |
| Topbar PointBadge | `/dashboard` | 클릭 | 통계 진입 |

## 11. Accessibility (KWCAG 2.2 Lite)

| 항목 | 정책 |
|------|------|
| 명도 대비 | 본문 텍스트 ≥ 4.5:1 (WCAG AA) |
| 키보드 탐색 | 모든 interactive 요소 `Tab` 순회 가능, focus ring 명확 |
| Article 본문 | 최소 16px, line-height 1.7 |
| Reduced motion | `prefers-reduced-motion` 감지 시 잎사귀 모션·트랜지션 disable |
| Semantic HTML | `<article>`, `<nav>`, `<main>`, `<aside>` 정확히 사용 |
| Alt text | Tree 이미지·일러스트에 의미 있는 alt |
| Skip-to-content | Topbar 다음 위치에 hidden link 제공 |

> **Note:** Phase 2에서 KWCAG 인증 검토.
